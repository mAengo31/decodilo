# Lambda Same Shape Capacity Retry Acceptance

Same-shape capacity retry acceptance is a separate future-review artifact. It
is not generated by default and generic catalog-only risk acceptance is not
enough to retry a shape that recently failed due to capacity.

The operator must acknowledge:

- the same shape recently failed due to capacity
- another attempt may return the same capacity error
- no automatic launch retry is allowed
- only one instance may be attempted
- no instance may be created if capacity is unavailable
- owned-instance termination is required if an instance is created
- termination verification is required
- OS shutdown is not sufficient
- the acceptance is for future review only, not immediate launch

Allowed statuses are `not_provided`,
`accepted_for_future_same_shape_capacity_retry_review`, and `declined`.
Forbidden statuses include `launch_now`, `launch_ready`, `launch_allowed`, and
`execute_now`.
